# ZXManage  本地服务运行模块使用文档

## 一、功能概述
**本地服务运行模块** 用于管理项目的本地服务实例，基于「配置模块」中预设的服务配置，自动展示可操作的服务列表。你可以直观查看服务状态，并对服务执行**启动、重启、停止**等操作，同时获取服务访问地址，目前只支持vue、react项目。

## 二、界面说明
### 模块入口
在项目详情页的「主页」标签下，找到 **本地服务运行** 区域。

![本地服务运行模块界面](/service.png)
<p style="text-align: center; color: #666;font-size:12px">图1：本地服务运行模块主界面</p>

### 核心元素
| 元素 | 说明 |
|------|------|
| 服务名称 | 展示配置的服务环境名称（如：测试） |
| 访问地址 | 服务运行后的可访问地址（如：`http://192.168.170.1:8080`），点击可复制 |
| 状态提示 | 顶部黄色提示框：提醒「安装路径含中文会导致服务运行失败」，启动失败时可优先排查 |
| 操作按钮 | 绿色启动按钮、灰色重启按钮、灰色停止按钮，按钮状态随服务运行状态变化 |


## 三、操作指南

### 1. 查看服务状态
- 服务未运行：**启动按钮** 为绿色可点击状态，重启/停止按钮为灰色不可用状态。
- 服务运行中：**启动按钮** 变为不可用状态，**重启/停止按钮** 变为可点击状态。

### 2. 启动服务
- 确认安装路径**无中文字符**（避免启动失败）。
- 点击目标服务对应的 **绿色启动按钮**。
- 等待服务启动完成，按钮状态将更新，访问地址变为可访问状态。

### 3. 重启服务
- 确保服务处于**运行中**状态。
- 点击 **重启按钮**，系统将先停止当前服务，再自动重新启动。
- 重启过程中按钮会短暂不可用，重启完成后恢复正常状态。

### 4. 停止服务
- 确保服务处于**运行中**状态。
- 点击 **停止按钮**，服务进程将被终止。
- 停止后，按钮状态恢复为「可启动」状态，访问地址暂时失效。


## 四、注意事项
- **路径中文问题**：安装路径、代码目录或项目路径中若包含中文字符，会直接导致服务启动失败。若启动失败，请先检查所有路径是否为纯英文/数字/符号。
- **依赖前置**：确保项目所需的打包运行环境（如 Node.js 等）已正确安装并配置到系统环境变量中，否则服务无法启动。
- **端口占用**：若启动失败提示「端口已被占用」，请在「配置模块」中修改服务端口，或关闭占用该端口的其他程序后重试。


## 五、常见问题排查
| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 点击启动后无响应/提示失败 | 路径含中文、依赖未安装、端口被占用 | 1. 检查所有路径是否为纯英文<br>2. 确认运行环境依赖已安装<br>3. 更换服务端口 |
| 服务启动后访问地址无法打开 | 服务未完全启动、防火墙拦截、地址错误 | 1. 等待几秒后重试访问<br>2. 检查系统防火墙是否放行该端口<br>3. 确认配置的IP/端口是否正确 |
| 重启/停止按钮灰色不可点击 | 服务当前未处于运行状态 | 先点击「启动」按钮，待服务运行后再操作 |
