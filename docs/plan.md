# 开发计划

ZXManage 的目标是成为开发者桌面上最得心应手的全能工具箱。我们不仅仅满足于基础的项目管理，更希望通过深入集成开发工作流，提升每一位开发者的效率。

> **产品定位**：专为开发者打造的本地项目全生命周期管理工具，解决多项目混杂、英文目录难识别、开发工具碎片化痛点，实现项目统一管控、高效管理。
> 我们的初衷很简单：让每一个英文命名的项目都有清晰归属，把零散的开发管理环节整合到一起，不用再费力回忆项目信息，一站式管好所有本地项目。

以下是我们的长期规划路线图，具体发布时间可能会根据开发进度进行调整。


## 🚀 近期计划 (v1.1.x) <Badge type="tip" text="开发中" />

我们聚焦核心体验升级，进一步提升工具使用效率，减少开发过程中的工具切换成本。

### 全局快捷键体系
- **功能描述**：为核心操作配置自定义快捷键，支持一键唤起常用功能。
- **核心快捷键示例**：
  - 项目快速搜索：`Ctrl/Cmd + K`
  - 新建项目笔记：`Ctrl/Cmd + N`
  - 启动本地服务：`Ctrl/Cmd + R`
- **扩展能力**：支持用户自定义快捷键映射，适配个人操作习惯。（目前还在考虑中是否增加自定义功能）

### 插件功能体系
- **功能描述**：把工具软件中需要用到的第三方插件做成管理模块，例如node、git等。

### 前端远程调试应用集成
- **功能描述**：新增前端远程调试专属应用模块，打通本地开发与远程调试链路。
- **核心能力**：
  - 远程调试：在打包时index.html插入远程调试代码,实现远程他人测试或者访问项目时记录日志（该阶段只实现内网与本地服务模块联动）
- **价值**：记录报错信息，回溯报错产生路径


## 🌟 中期规划 (v1.2.x) <Badge type="warning" text="规划中" />

聚焦效率升级，完善工具闭环能力，进一步降低开发流程中的人工操作成本。

### 版本自动更新机制
- **功能描述**：内置版本检测与自动更新模块，无需手动下载安装包。
- **核心特性**：
  - 静默更新检测：启动时自动检测新版本，不打扰当前操作
  - 增量更新：仅下载更新包差异内容，节省带宽和存储空间
  - 更新日志展示：清晰呈现每个版本的功能迭代和问题修复
- **价值**：让开发者始终使用最新、最稳定的版本，无需关注版本更新细节。

### 远程打包部署能力（未来规划，上云端服务后才可）
- **功能描述**：支持远程将本地项目一键打包并部署至远程服务器/测试环境。
- **核心流程**：
  1. 工具集成云端服务，开启‌WebSocket功能，小程序/H5远程触发打包部署
  2. 实时监听是否收到打包命令
  3. 收到命令后一键执行「本地打包 + 产物上传 + 远程部署」
- **价值**：git是内网时人在外面也可实现远程打包无需返回公司操作
- **安全性考虑**：该功能存在安全性问题目前还在考虑是否上架该功能


## 🏗️ 远期愿景 (v2.0.0+)

构建完整的ZXManage生态体系，从"工具"升级为"开发者工作台"。

### 插件生态完善
- **功能描述**：开放完整插件API，打造插件市场，支持插件分发与管理。
- **核心目标**：
  - 第三方开发者可发布共享插件
  - 插件评分与审核机制，保障插件质量
  - 插件依赖管理，支持插件间协同工作
- **场景拓展**：覆盖全栈开发全流程，从项目创建到部署上线的全链路插件支持。

### 全栈开发闭环
- **功能描述**：深度整合前后端开发能力，实现真正的一站式开发管理。
- **核心升级**：
  - 后端服务调试集成：支持Java/PHP/Node等后端服务的本地调试
  - 数据库可视化管理：集成MySQL/Redis/MongoDB等数据库管理能力
  - 接口调试与文档：内置接口测试工具，自动生成接口文档
- **价值**：在单一工具内完成全栈开发的所有核心操作。

### 团队协作能力
- **功能描述**：新增团队协作模块，支持项目配置、开发规范、部署流程的团队共享。
- **核心能力**：
  - 团队项目库：统一管理团队所有项目，权限精细化控制
  - 开发规范同步：团队级代码规范、提交规范统一配置
  - 协作笔记：支持项目笔记的团队共享与评论
- **前提**：保持本地私有化存储核心特性，协作数据可选择本地/云端存储。
